  • ¿Cuál es la diferencia entre Internet y la World Wide Web?

    • Respuesta:

  • ¿Cuáles son las partes de un URL?

    • Respuesta:

    Las partes de un URL son: el protocolo, el dominio, la ruta y los parámetros

  • ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

    • Respuesta:

    • GET: Recuperar información de un recurso
    • HEAD: Obtener solo los encabezados de una solicitud GET
    • POST: Enviar datos al servidor para crear o actualizar un recurso
    • PUT: Actualizar un recurso existente en el servidor
    • PATCH: Aplicar modificaciones parciales a un recurso
    • DELETE: Eliminar un recurso del servidor

  • ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

    • Respuesta:

    Un atributo HTML5 está depreciado o desaprobado (deprecated) cuando se ha marcado como obsoleto, su uso no es recomendado y puede ser eliminado en futuras versiones de HTML. Algunos elementos de HTML 4 que están desaprobados en HTML5 son:

    • font: Este elemento se utilizaba para definir el estilo de texto, como el tipo de letra, tamaño y color. En HTML5, se recomienda utilizar CSS para controlar la apariencia del texto.
    • center: Este elemento se utilizaba para centrar el contenido. En HTML5, se recomienda utilizar CSS para centrar elementos en lugar de este elemento obsoleto.
    • marquee: Este elemento se utilizaba para crear texto o imágenes en movimiento. En HTML5, se recomienda evitar su uso debido a problemas de accesibilidad y usabilidad.

  • ¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

    • Respuesta:

    • Nuevos elementos semánticos: HTML5 introdujo nuevos elementos como header, footer, article, section, entre otros, que mejoran la estructura y semántica del contenido web.
    • Soporte multimedia: HTML5 incluye elementos nativos para audio y video (audio y video), lo que facilita la integración de contenido multimedia sin necesidad de plugins externos.
    • Mejoras en formularios: HTML5 introdujo nuevos tipos de entrada para formularios, como email, date, number, entre otros, lo que mejora la validación y la experiencia del usuario al completar formularios.
    • APIs adicionales: HTML5 incluye varias APIs nuevas, como la API de geolocalización, la API de almacenamiento local, la API de arrastrar y soltar, entre otras, que permiten a los desarrolladores crear aplicaciones web más interactivas y funcionales.

  • ¿Qué componentes de estructura y estilo tiene una tabla?

  • Sobre el ciclo de vida y desarrollo de los sistemas de información:

    1. ¿Cuál es el ciclo de vida de los sistemas de información?

      2. Respuesta:

      El ciclo de VIDA de los sistemas de información consta de las siguientes fases:

      • Planificación: En esta fase se identifican las necesidades del sistema, se establecen los objetivos y se define el alcance del proyecto.
      • Análisis: Se recopilan y analizan los requisitos del sistema, se identifican los procesos y se crean modelos de datos.
      • Diseño: Se diseña la arquitectura del sistema, se definen las interfaces de usuario y se crean los diagramas de flujo.
      • Desarrollo: Se lleva a cabo la programación y construcción del sistema según el diseño establecido.
      • Pruebas: Se realizan pruebas para verificar que el sistema cumple con los requisitos y funciona correctamente.
      • Implementación: Se despliega el sistema en el entorno de producción y se capacita a los usuarios finales.
      • Mantenimiento: Se realizan actualizaciones, correcciones de errores y mejoras continuas al sistema para garantizar su funcionamiento a largo plazo.

    2. ¿Cuál es el ciclo de DESARROLLO de sistemas de información?

      3. Respuesta:

      El ciclo de desarrollo de sistemas de información es un proceso iterativo que incluye las siguientes etapas:

      • Recolección de requisitos: Se recopilan y documentan los requisitos del sistema a través de entrevistas, encuestas y análisis de procesos.
      • Análisis de requisitos: Se analizan los requisitos recopilados para identificar las funcionalidades necesarias y las restricciones del sistema.
      • Diseño del sistema: Se crea un diseño detallado del sistema, incluyendo la arquitectura, la estructura de datos y las interfaces de usuario.
      • Desarrollo: Se lleva a cabo la programación y construcción del sistema según el diseño establecido.
      • Pruebas: Se realizan pruebas para verificar que el sistema cumple con los requisitos y funciona correctamente.
      • Implementación: Se despliega el sistema en el entorno de producción y se capacita a los usuarios finales.
      • Mantenimiento: Se realizan actualizaciones, correcciones de errores y mejoras continuas al sistema para garantizar su funcionamiento a largo plazo.
